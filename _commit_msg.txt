feat(builder): Introduce dialect-specific query compilation

This commit introduces the basic infrastructure for compiling SQL queries with
dialect-specific syntax. If not specified, we will default to more or less just
ANSI SQL without quoting

Specifically, this commit:

  - Adds a `Dialect` enum to represent different SQL dialects (Presto, MSSQL, etc.)
  - Creates a `Compiler` base class and a `PrestoCompiler` to handle dialect-specific SQL generation
  - Adds a `set_dialect` method to the `Sustained` model to configure the dialect for queries
  - Updates the `QueryBuilder` class and all clause builders to be aware of the configured dialect and use the compiler to generate SQL
